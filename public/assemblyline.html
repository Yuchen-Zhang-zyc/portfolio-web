Project Overview

What is Assembly Line Y?

01

VR Factory Training

Players step inside a virtual Tesla Model Y assembly environment and install 16 interior components across 8 guided steps — from pedals to door panels — in the correct sequence.
02

AI-Guided Experience

ARIA, an AI assistant built on Claude API, provides real-time voice guidance, answers natural language questions, and adapts to the trainee's progress throughout the session.
03

Skill Evaluation

Performance is scored across four tiers — Rookie, Apprentice, Expert, Master — based on sequence accuracy and interaction quality, providing actionable feedback at session end.
Background Research

AR/VR in Automotive Assembly

Automotive interior assembly remains one of the most labor-intensive stages of vehicle production. Unlike engine assembly — increasingly handled by robotic systems — interior fitting depends heavily on human dexterity, spatial judgment, and sequenced procedure. This gap makes it an ideal target for immersive training technology.

Industry Precedent

A Growing Industry Standard

The automotive AR/VR market was valued at $21.3 billion in 2024 and is projected to reach $115.8 billion by 2032 — a CAGR of 26.6%. Platforms like Scope AR's WorkLink and PTC's Vuforia Expert Capture are already deployed on real factory floors, overlaying step-by-step instructions onto physical components. Volkswagen's AR-driven training programs have been shown to reduce production errors by 30%, while studies indicate AR-guided training helps new hires reach expert assembly speeds up to 40% faster. Assembly Line Y applies this same principle in a fully virtual, zero-risk environment — no physical hardware required on the factory floor.
Design Decision

Why Interior Assembly?

Engine assembly and battery pack installation are increasingly automated — welding robots and conveyor systems handle the majority of powertrain work. Interior assembly, however, still requires human hands: fitting seat rails, routing wiring harnesses, aligning dashboard panels. The manual nature of this stage makes it both training-critical and well-suited to an immersive, hands-on VR simulation.
Key Features

Project Highlights

01

Natural Voice Interaction

Players speak naturally to ARIA using Wit.ai speech recognition. No keyword commands — Claude API understands intent, making the training feel conversational rather than scripted.
02

Orientation-Aware Snap Sockets

Parts must be oriented correctly before they snap into place. Angle tolerance is enforced at the socket level, reflecting real assembly constraints where direction matters.
03

Scanner Gun Raycasting

A scanner gun uses raycasting to identify and confirm parts before installation — adding a realistic process step inspired by actual factory barcode scanning workflows.
04

Multi-Modal Interactions

Grab, poke, press, and voice — each part type uses the interaction that fits it best. Steering wheel requires a poke to lock in, airbag requires press-confirm, doors require careful placement.
05

Wrist HUD Interface

A world-space wrist display shows assembly progress, current step, and ARIA's latest message. Toggle with B button. Designed with a Lazy Follow system to stay comfortably readable.
06

Guided Lobby System

A dedicated lobby scene uses ARIA to guide players through four areas before training begins. ARIA detects intent to progress and triggers area lighting and scene transitions automatically.
Technical

System Architecture

Layer 1 — Input

Voice & Controller Input

VoiceInputManager.cs captures speech via Wit.ai on right-hand A button hold. XR controller bindings handle grab (grip), poke (trigger), scanner confirm (X), HUD toggle (B), and jump (Y) — all bound in code, no Inspector wiring.
Layer 2 — AI Dialogue

ARIA + Claude API

ARIAController.cs orchestrates the dialogue pipeline. Transcribed speech passes to ClaudeAPIManager.cs which calls claude-haiku-4-5 with a dynamic system prompt containing the current assembly context. Responses are parsed for ACTION tags before TTS playback via Meta Voice SDK.
Layer 3 — State

Game State & Assembly Logic

GameStateContext.cs acts as single source of truth. AssemblyManager.cs listens to socket events from 16 parts across 8 steps, validates sequence, and updates state. ScoreManager.cs tracks timing and accuracy to compute the final tier rating.
Layer 4 — Interaction

XR Interaction Layer

LockableGrabInteractable.cs extends Unity's XRI grab system. OrientationSocket.cs validates rotation before accepting a snap. ScannerGun.cs fires raycasts to identify parts. PartHighlighter.cs and PartAudioHandler.cs provide visual and audio feedback on every interaction.
Layer 5 — UI

HUD & Display

WristUIManager.cs drives a World Space Canvas on the left wrist with Lazy Follow interpolation and wrist-raise detection. LobbyManager.cs manages lobby state, area lighting sequences, and scene transition with screen fade — all triggered by ARIA's ACTION tags.

AI System

Meet ARIA

ARIA
ADAPTIVE REAL-TIME INTELLIGENT ASSISTANT

ARIA is the AI core of Assembly Line Y. Powered by Claude API (claude-haiku-4-5), she provides real-time training guidance, answers natural language questions, and adapts her system prompt dynamically as the assembly progresses through each of the 8 steps.


In the lobby, ARIA also acts as the environment guide — detecting intent from free-form speech and triggering area lighting and scene transitions through structured ACTION tags in her responses.

01

Player holds A + speaks

Wit.ai captures audio and transcribes to text in real time
02

Context injected

Current step, parts installed, and score appended to system prompt
03

Claude API called

claude-haiku-4-5 responds with [ACTION:xxx] tag + natural language
04

ACTION parsed

LobbyManager or AssemblyManager handles NEXT / START / HINT / NONE
05

TTS playback

Cleaned response text spoken aloud via Meta Voice SDK (Voice: Aria)
